# Study Notes: Software Testing Tools and Measurement

## 1. Manual Testing
**Short Explanation:** A process where a human tester checks the software without automation tools. The tester acts as an end-user to verify features and find defects by following test cases step-by-step.

### Limitations of Manual Testing
*   **Time & Speed:** Very time-consuming and slow compared to automation.
*   **Repetition:** Inefficient for repetitive tasks like regression testing.
*   **Accuracy:** Prone to human errors.
*   **Coverage:** Hard to test every possible input and scenario.
*   **Resources:** Requires more manpower, increasing project costs.
*   **Performance:** Not reliable for measuring load or stress.

### The Manual Testing Process (8 Steps)
1.  **Requirement Analysis:** Understand what to test by looking at user stories or SRS.
2.  **Test Planning:** Decide strategy, scope, timeline, and resources.
3.  **Test Case Design:** Write specific test cases and prepare test data.
4.  **Test Environment Setup:** Prepare the hardware and software for testing.
5.  **Test Execution:** Run tests manually and mark them as Pass/Fail.
6.  **Defect Reporting:** Log bugs in a tracking tool with severity and priority.
7.  **Re-testing & Regression:** Test fixed bugs and ensure existing features still work.
8.  **Test Closure:** Prepare summary reports and sign off.

---

## 2. Automation Testing
**Short Explanation:** Using software tools (like Selenium or JUnit) to run test scripts automatically. It compares actual results with expected results to improve speed and reliability.

### Benefits and Importance
*   **Efficiency:** Faster execution and saves time/cost in the long run.
*   **Reliability:** Highly accurate and can run 24/7.
*   **Reusability:** Scripts can be used multiple times for regression testing.
*   **Human Resources:** Requires fewer people for execution.

### The Automation Testing Process (6 Steps)
1.  **Feasibility Analysis:** Decide if automation is suitable based on cost and stability.
2.  **Tool Selection:** Choose a tool based on technology, budget, and team skill.
3.  **Evaluate Framework:** Select a structure (Data-driven, Keyword-driven, or Hybrid).
4.  **Proof of Concept (POC):** Create a small sample to verify the tool's capability.
5.  **Develop Framework:** Build the structure for scripts, reporting, and error handling.
6.  **Develop & Execute Scripts:** Write the code, run tests, and analyze failures.

---

## 3. Comparison: Manual vs. Automation

| Aspect | Manual Testing | Automation Testing |
| :--- | :--- | :--- |
| **Execution** | By humans (simulating users) | By tools (using scripts) |
| **Speed** | Slower | Faster |
| **Accuracy** | Prone to human error | Highly consistent |
| **Cost** | High long-term cost | High initial cost; cheap long-term |
| **Programming** | Not required | Programming knowledge required |
| **Best Use** | UI/UX, Exploratory testing | Regression, Load testing |

---

## 4. Software Testing Tool Categories
*   **Manual/Defect Tracking:** JIRA, Bugzilla, Mantis.
*   **Automation (Functional):** Selenium, UFT (QTP), Cypress.
*   **Performance:** JMeter, LoadRunner.
*   **Mobile:** Appium, Espresso.
*   **API:** Postman, SoapUI.
*   **Security:** OWASP ZAP, Burp Suite.

---

## 5. Selecting a Testing Tool
**Short Explanation:** Choosing the right tool is critical because tools can be expensive, require specific training, or may not support all platforms.

### Selection Criteria
*   **Meeting Requirements:** Does it support the project goals and types of testing?
*   **Technology Expectations:** Is it compatible with the project’s language, OS, and browsers?
*   **Training and Skill:** Can the current team learn and use the tool effectively?
*   **Management Aspects:** What is the cost, license type, and Return on Investment (ROI)?

---

## 6. Selenium
**Short Explanation:** A free, open-source tool used specifically for automating web applications. It is platform-independent and supports multiple languages.

### Key Features
*   **Cross-Browser:** Works on Chrome, Firefox, Edge, and Safari.
*   **Multi-Language:** Supports Java, Python, C#, and Ruby.
*   **Parallel Execution:** Can run multiple tests at once to save time.
*   **Open Source:** No licensing costs.

### Selenium Components
*   **Selenium IDE:** A "record-and-playback" tool for beginners to create quick scripts.
*   **Selenium WebDriver:** The core component for complex testing; communicates directly with browsers.
*   **Selenium Grid:** Used to run tests on multiple machines and operating systems simultaneously.
