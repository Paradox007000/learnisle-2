### **1. Project Scheduling: Overview**
Project scheduling is the process of organizing project work into separate tasks and deciding when and how they will be executed.

*   **Definition:** A mechanism to communicate tasks, resource allocation, and timeframes.
*   **Purpose:** To track deadlines, start/end dates, milestones, and task dependencies.
*   **Resources Involved:** People, equipment, software tools, budget, and materials.

---

### **2. The 5-Step Project Scheduling Process**
1.  **Identify Activities:** Break the project into small, manageable tasks with clear start and end points (e.g., coding, testing).
2.  **Identify Activity Dependencies:** Determine the order of tasks.
    *   **Finish-to-Start (FS):** Task B starts after Task A finishes.
    *   **Start-to-Start (SS):** Task B starts when Task A starts.
    *   **Finish-to-Finish (FF):** Task B finishes when Task A finishes.
    *   **Start-to-Finish (SF):** Task B finishes when Task A starts (Rare).
3.  **Allocate Resources:** Assign tools, money, and materials. Ensure they are available to avoid delays.
4.  **Create Project Charts:** Use visual tools like **Gantt Charts** (timelines) or **PERT/CPM Charts** (critical paths).
5.  **Allocate People:** Assign team members based on skills and availability. Avoid overloading individuals.

---

### **3. Work Breakdown Structure (WBS)**
A WBS is a hierarchical "tree" decomposition of a project into smaller parts.

*   **Approach:** Top-Down (starts with the project name and breaks down into leaf-level independent activities).
*   **WBS Levels:**
    *   **Level 1:** Project Goal (The complete system).
    *   **Level 2:** Major Modules (e.g., Design, Implementation, Testing).
    *   **Level 3:** Sub-Modules/Activities (e.g., Database design, Unit testing).
    *   **Level 4:** Work Packages (The smallest manageable task assigned to a person).

---

### **4. Activity Networks**
These are graphical representations of project tasks showing their logical sequence and dependencies.

*   **AON (Activity on Nodes):** Activities are shown as circles or ovals; arrows show the sequence.
*   **AOA (Activity on Arrows):** Activities are shown as arrows; nodes represent the start or end points (events).

---

### **5. Project Tracking & Visualization**
**Project Tracking** is the process of monitoring progress to ensure the project stays on schedule, within budget, and meets quality standards.

*   **Gantt Chart (Timeline Chart):** 
    *   Uses horizontal bars to show the start and end dates of tasks.
    *   Helps visualize the sequence, duration, and progress of work.
*   **Timeline Chart Purpose:** Identifies overlaps, helps adjust resources, and captures a sequence of events.

---

### **6. Earned Value Analysis (EVA)**
EVA is a technique used to measure project performance regarding cost and schedule.

**Key Concepts:**
*   **Planned Value (PV):** Budgeted cost of work planned to be done by a specific time.
*   **Earned Value (EV):** Budgeted cost of work *actually* completed.
*   **Actual Cost (AC):** The real cost spent to complete the work.

**Performance Metrics:**
*   **Schedule Variance (SV = EV - PV):** If negative, the project is **behind schedule**.
*   **Cost Variance (CV = EV - AC):** If negative, the project is **over budget**.
*   **Schedule Performance Index (SPI = EV / PV):** If less than 1, the project is **behind schedule**.
*   **Cost Performance Index (CPI = EV / AC):** If less than 1, the project is **over budget**.
