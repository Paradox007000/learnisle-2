### Software Testing Life Cycle (STLC) Study Notes

The **Software Testing Life Cycle (STLC)** is a sequence of specific activities performed during the testing process to ensure software quality.

---

### The 6 Phases of STLC

#### 1. Requirement Analysis
*   **Explanation:** Testers study the software requirements to understand what needs to be tested.
*   **Key Activities:**
    *   Identify testable requirements.
    *   Prepare the Requirement Traceability Matrix (RTM).
*   **Example:** For a login page, the tester identifies scenarios like valid login, invalid login, and blank fields.

#### 2. Test Planning
*   **Explanation:** Defining the strategy, resources, and schedule for testing.
*   **Key Activities:**
    *   Create a Test Plan.
    *   Determine testing scope, tools (e.g., Selenium), and team resources.
    *   Estimate the effort, cost, and duration.
*   **Example:** Deciding that testing will take 2 weeks with a team of 3 testers.

#### 3. Test Case Design / Development
*   **Explanation:** Creating the detailed steps and data needed for testing.
*   **Key Activities:**
    *   Write specific test cases.
    *   Prepare necessary test data.
    *   Review the test cases for accuracy.
*   **Example:** Writing a test case to verify that entering a valid email and password results in a successful login.

#### 4. Test Environment Setup
*   **Explanation:** Preparing the hardware and software "lab" where the testing will happen.
*   **Key Activities:**
    *   Install the application.
    *   Set up test servers, databases, and browsers (e.g., Chrome, Firefox).
*   **Example:** Connecting the shopping app to a test server and database.

#### 5. Test Execution
*   **Explanation:** The actual process of running the tests and finding bugs.
*   **Key Activities:**
    *   Execute the designed test cases.
    *   Compare the actual results against the expected results.
    *   Log defects (bugs) if the results do not match.
*   **Example:** Finding that an error message does not appear when a wrong password is used and logging it as a defect.

#### 6. Test Closure
*   **Explanation:** Wrapping up the testing cycle and documenting the final results.
*   **Key Activities:**
    *   Prepare a Test Summary Report.
    *   Analyze the final test results (Pass/Fail counts).
    *   Discuss lessons learned during the process.
*   **Example:** Reporting that out of 120 test cases, 110 passed and 10 failed, with major defects fixed before release.
