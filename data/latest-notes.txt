### **1. Project Scheduling**
**Definition:** A process used to track tasks, deadlines, and resource allocation within a project timeframe.

*   **Key Functions:**
    *   Communicates tasks and start/end dates.
    *   Identifies necessary resources (people, equipment, budget).
    *   Shows dependencies (how tasks rely on one another).

**The 5-Step Scheduling Process:**
1.  **Identify Activities:** Break the project into small, manageable tasks (e.g., design, coding).
2.  **Identify Activity Dependencies:** Determine the order of tasks.
    *   *Finish-to-Start (FS):* Task B starts only after Task A finishes.
    *   *Start-to-Start (SS):* Task B starts when Task A starts.
    *   *Finish-to-Finish (FF):* Task B finishes only when Task A finishes.
    *   *Start-to-Finish (SF):* Task B finishes only when Task A starts (Rare).
3.  **Allocate Resources:** Assign tools, software, and budget to each activity.
4.  **Create Project Charts:** Use visual tools like Gantt Charts (timelines) or PERT/CPM Charts (critical paths).
5.  **Allocate People:** Assign team members based on skills without overloading them.

---

### **2. Work Breakdown Structure (WBS)**
**Definition:** A "Top-Down" hierarchical tree that breaks a complex project into smaller, independent parts.

*   **Structure Levels:**
    *   **Level 1 (Project Goal):** The entire system (e.g., Library Management System).
    *   **Level 2 (Major Modules):** Requirements, Design, Implementation, Testing, etc.
    *   **Level 3 (Sub-Modules):** Specific activities like "Database Design" or "Unit Testing."
    *   **Level 4 (Work Packages):** The smallest manageable tasks assigned to individuals.

---

### **3. Activity Networks**
**Definition:** A graphical map of project tasks showing their logical sequence and flow.

*   **Two Representation Methods:**
    1.  **Activity on Node (AON):** Activities are shown in circles/ovals. Arrows show the sequence.
    2.  **Activity on Arrow (AOA):** Activities are represented by arrows. Nodes (circles) represent the start or end points of an activity.

---

### **4. Project Tracking Tools**
**A. Timeline / Gantt Chart**
*   **What it is:** A bar chart showing horizontal lines that represent the duration of tasks against a timeline.
*   **Purpose:** To visualize the schedule, identify overlaps, and monitor deadlines.

**B. Earned Value Analysis (EVA)**
*   **What it is:** A technique to measure project health by integrating cost, schedule, and scope.
*   **Key Metrics:**
    *   **Planned Value (PV):** Budgeted cost for work *planned* to be done.
    *   **Earned Value (EV):** Budgeted cost for work *actually* completed.
    *   **Actual Cost (AC):** The real money spent so far.
*   **How to check project health:**
    *   **Schedule Variance (SV = EV - PV):** If negative, you are behind schedule.
    *   **Cost Variance (CV = EV - AC):** If negative, you are over budget.
    *   **SPI (EV / PV):** If less than 1.0, the project is behind schedule.
    *   **CPI (EV / AC):** If less than 1.0, the project is over budget.
